Remarks

Exhaustively testing all possible inputs to any nontrivial software component is generally impossible due to the enormous number of variations. Combinatorial testing is one approach to achieve high coverage with a much smaller set of variations. Pairwise, the most common combinatorial strategy, test every possible pair of values. Higher orders of combinations (3-wise, 4-wise, etc.) can also be used for higher coverage at the expense of more variations. See Pairwise Testing and Pairwise Testing in Real World for more resources.
